Detailed Description

The AllocatorStorage concept defines a generalized way of storing allocators within composite types.

The requirements are minimal – only requiring a 'get_allocator' function that returns a reference to the underlying allocator, which must itself satisfy the Allocator concept

For type S to be AllocatorStorage, it must satisfy the following:

Provided

  • S - an AllocatorStorage type
  • s - an instance of type S
  • cs - an instance of type const S
  • A - the type of the underlying stored allocator
  • a - an instance of type A

the following expressions must be well-formed with the expected side-effects:

S::allocator_type

Retrieves the underlying allocator type. allocator_type must satisfy the Allocator concept.

A& a = s.get_allocator();

Retrieves reference to the underlying stored allocator.

const A& a = cs.get_allocator();

Retrieves reference to the underlying stored allocator.
