Detailed Description

This concept defines a manner of tracking allocations and deallocations to determine leaks and double-deletions.

For type T to be MemoryTracker, it must satisfy the following:

Provided

  • T - a MemoryTracker type
  • t - an instance of type T
  • s - the size of an allocation
  • a - the alignment of the allocation
  • v - a void pointer
  • i - an allocator_info object

The following expressions must be well-formed and with the following side-effects:

t.on_allocate( p, s, a );

T receives information indicating that p was allocated with size s and alignment a

t.on_deallocate( info, p, s );

T receives information indicating that p was deallocated and size requested was s, for the allocator indicated by info i

t.on_deallocate_all();

T receives information indicating the all memory was deallocated from within the allocator

t.finalize( i );

T receives notification that all memory tracking has been completed. It is at this time that finalization of the tracking may occur (e.g. determining global memory leaks)
